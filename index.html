DSC 106 Final Project Prototype

Road to Sustainability: Understanding Vehicle Efficiency and CO₂ Emissions

Team Members: Siya Sathaye, Juee Deshmukh, Sahar Zahir, Arushi Patra

This prototype explores how everyday vehicle choices shape fuel efficiency and carbon emissions. We focus on the tradeoff between mileage and pollution output so readers can quickly compare vehicle groups instead of scanning raw tables. The broader goal is to turn transportation sustainability into something concrete, visual, and easier to reason about.

Mileage vs. CO₂ Emissions

Each circle represents one recorded trip. Higher mileage is better, while lower CO₂ per kilometer is better. Use the filter to compare how different vehicle classes cluster.

What have we done so far?

We have built an interactive scatter plot visualizing the tradeoff between actual mileage (km per liter) and CO₂ emissions (g per km) across 600 recorded vehicle trips. Each data point is color-coded by fuel type: Diesel, Petrol, and Hybrid, making it easy to spot how different fuel technologies cluster in the efficiency-emissions space. In addition to that, a dropdown filter was implemented that lets readers isolate specific vehicle classes (Hatchback, Luxury, MUV, SUV, Sedan) to compare how each category performs. We added notes for users to understand that higher mileage and lower C02 emissions are indicators of high efficiency.

Design Challenges

The most challenging aspect will be designing a narrative flow that guides readers through the data without overwhelming them, balancing interactivity with storytelling so the page feels like an guided explanation rather than just a dashboard with pasted charts. We plan to add additional visualizations (such as breakdowns by engine size or manufacturer), and linking these charts together so interactions in one update the others will require careful state management in D3. Another challenge is making the annotations and callouts meaningful: pointing out surprising findings (like where hybrids underperform expectations) rather than just labeling obvious clusters. With only 600 rows to work with, we need to be strategic about which columns and relationships to highlight in order to craft a well thought out project that meets all the requirements. Finally, making the experience work well at different screen sizes while keeping the D3 layouts responsive will require extra attention to layout design.
